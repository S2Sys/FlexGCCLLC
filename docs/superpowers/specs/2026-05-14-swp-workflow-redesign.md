# SWP Workflow Redesign — Spec

**Date:** 2026-05-14
**Status:** Approved
**Author:** Senthilvel Thangavelu

---

## Summary

Restructure the SmartWorkz planning phase workflow to:
1. Enforce architecture before UI/DB design
2. Merge swp-arch + swp-scaffold into a two-stage command
3. Create BREAKDOWN.md at SRS sign-off and update it incrementally at every phase sign-off
4. Integrate subagent-driven-development for all design and dev phases

---

## New Phase Order

```
Phase 0:  /swp-validate  →  /swp-srs
Phase 1:  /swp-arch  (Stage 1: design → Stage 2: scaffold)
Phase 2:  /swp-ui  +  /swp-db  (parallel subagents, independent)
Phase 3:  /swp-plan  (unblocks when Phase 1 + 2 all complete)
Cross:    /swp-sync  (after each gate)
```

**Previous order:** srs → ui → db → arch → scaffold → plan
**New order:** srs → arch+scaffold → ui+db (parallel) → plan

## BREAKDOWN.md — Lifecycle

### Created at SRS sign-off

`swp-srs` post sign-off block creates `docs/BREAKDOWN.md` with a full skeleton:

```markdown
## Phase 1A — Architecture Design   [ ] pending
## Phase 1B — Scaffold              [ ] pending
## Phase 2A — UI/UX Design          [ ] pending
## Phase 2B — DB Design             [ ] pending
## Phase 3  — Dev Stories           [ ] BLOCKED (requires 1A+1B+2A+2B complete)
```

### Updated at each phase sign-off

Each command appends its section detail and marks `[x]` with completion date on sign-off.

```markdown
## Phase 1A — Architecture Design   [x] 2026-05-20
  - Layered architecture: API / Service / Repository
  - DI container: Microsoft.Extensions.DI
  - Pattern: BaseAuditableEntity, ITenantScoped

## Phase 1B — Scaffold              [x] 2026-05-20
  - Build: 0 errors, 0 warnings

## Phase 2A — UI/UX Design          [x] 2026-05-21
  - Screens: 12 defined
  - Dark mode: confirmed
  - WCAG: AA compliant

## Phase 2B — DB Design             [x] 2026-05-21
  - Tables: 8, SPs: 3
  - Migration plan: included

## Phase 3  — Dev Stories           [ ] in progress
  - [ ] Story 1: ...
  - [x] Story 2: ...
```

### Phase 3 guard

`swp-plan` reads BREAKDOWN.md on start. If any of §1A, §1B, §2A, §2B are not `[x]`, it exits with:

```
BLOCKED: Phase [N] not complete. Run /swp-[command] first.
```

## swp-arch Redesign — Two-Stage Command

Single command `/swp-arch`, two approval gates:

```
Stage 1: Architecture Design
  - Input:    SRS.md
  - Output:   docs/ARCH-DESIGN.md, updated docs/ENTITIES.md
  - Subagent: arch design subagent dispatched
  - Review 1: spec reviewer — does ARCH-DESIGN.md cover all SRS requirements?
  - Review 2: quality reviewer — layers, DI, patterns correctly defined?
  - Gate:     human sign-off
  - On pass:  BREAKDOWN.md §1A = [x], auto-continues to Stage 2

Stage 2: Scaffold
  - Input:    SRS.md + ARCH-DESIGN.md
  - Output:   scaffolded project (zero build errors, zero warnings)
  - Subagent: scaffold subagent dispatched
  - Review 1: spec reviewer — does scaffold match ARCH-DESIGN.md structure?
  - Review 2: quality reviewer — build verification (zero errors confirmed)
  - Gate:     human sign-off
  - On pass:  BREAKDOWN.md §1B = [x]
```

`swp-scaffold.md` is marked internal — called only by swp-arch Stage 2, not standalone.

## Subagent-Driven-Development Integration

### Phase 1 — Sequential subagents
Architecture must complete before scaffold. Scaffold must complete before UI/DB.

```
Task 1A → implementer → spec reviewer → quality reviewer → sign-off
Task 1B → implementer → spec reviewer → quality reviewer → sign-off
```

### Phase 2 — Parallel design subagents
UI and DB are independent (write different docs, no shared state).

```
Task 2A (UI)  ┐  dispatched in parallel
Task 2B (DB)  ┘
Each: implementer → spec reviewer → quality reviewer → sign-off
```

### Phase 3 — Full SDD per dev story
```
/swp-plan creates story tasks in BREAKDOWN.md
Each story: implementer → spec reviewer → quality reviewer → BREAKDOWN §3 story [x]
Final: code reviewer → /finishing-a-development-branch
```
